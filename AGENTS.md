# AGENTS.md

## Repository notes for agents

**pufbx** is a Python/Cython binding for the [ufbx](https://github.com/ufbx/ufbx) FBX file loader. It is a single-package library (not a monorepo) with no external services, databases, or network dependencies.

### Prerequisites

- **Python 3.12** with dev headers (`python3.12-dev`) â€” needed to compile the Cython/C extensions.
- **gcc** â€” the vendored `ufbx/ufbx.c` is compiled during install.
- **uv** â€” the project's preferred package manager (see `justfile`).

### Build & install (editable, dev)

From the repo root, either:

```
uv sync --python 3.12 --extra dev
```

or the classic venv flow:

```
uv venv --python 3.12 .venv
source .venv/bin/activate
uv pip install -e ".[dev]"
```

On Windows, activate with `.venv\Scripts\activate` instead of `source`.

The editable install compiles ~51 Cython `.pyx` modules and the C library. Expect the first build to take ~1 minute.

### Running tests

```
uv run python -m pytest tests/ -v
```

All tests under `tests/` should pass. The single test fixture lives at `tests/fixtures/cube_and_bone.fbx` (generated by `tests/utils/generate_test_file.py` via Blender).

GitHub Actions runs the suite on **Ubuntu, macOS, and Windows** for **Python 3.9â€“3.13** (see `.github/workflows/build.yml`). Lint stays on a single Python (3.11).

### Lint / format

The `justfile` uses `black`, `isort`, and `autopep8`. Note: the existing codebase has some pre-existing lint issues that are not regressions.

```
uv run black --check pufbx/ tests/
uv run isort --check pufbx/ tests/
```

### Gotchas

- `.python-version` is **3.12** (local default for `uv`). CI still tests **3.9â€“3.13** on all platforms in the matrix; lint uses 3.11.
- The build requires `python3.12-dev` (or equivalent) for `Python.h`. Without it the C extension compilation fails.
- On Linux/macOS, `ufbx_wrapper` is loaded with `RTLD_GLOBAL` so all extension modules share C symbols. This is handled automatically by `pufbx/__init__.py`.
- On Windows, `ufbx.c` is built into a single extension module `pufbx._ufbx`; other native modules link against it so string interning stays consistent. `pufbx/__init__.py` imports `_ufbx` before `ufbx_wrapper`.
- `anim_to_array()` takes a filename string (not a `Scene` object) and returns a `(data, times, names)` tuple.
- `bake_anim()` takes a `Scene` object and returns a `BakedAnim` (not a dict); use `.modified_nodes` to iterate.

## Cursor Cloud specific instructions

The VM update script runs `uv sync --python 3.12 --extra dev` on startup, which installs all dependencies and compiles the Cython/C extensions. System prerequisites (`python3.12-dev`, `gcc`, `uv`) are pre-installed in the VM snapshot.

- **No external services** are needed. This is a standalone library; just install and run tests.
- **First build takes ~80s** because it compiles ~51 Cython extensions and the vendored C library. Subsequent `uv sync` calls are fast if nothing changed.
- **CI/CD for PyPI**: `.github/workflows/build.yml` handles lint, test matrix, wheel builds (cibuildwheel), and PyPI publishing. Publishing triggers on a GitHub release event and uses trusted publishing (OIDC via `id-token: write`). The repo owner must configure a trusted publisher on PyPI for `Owca-Xsido/pufbx`.
- Standard commands are documented in the `AGENTS.md` sections above and in `README.md`.
